我很高興伊薩富爾福德能和我一起教授這門課。她是 OpenAl 的技術人員，曾開發過流行的 ChatGPT 檢索插件，她的大部分工作是教導人們如何在產品中使用 LLM 或大型語言模型技術。她也為教人們提示的 OpenAl Cookbook 做出了貢獻。我很高興你能和我在一起。我很高興來到這裡與大家分享一些最佳實踐。因此，網路上有很多提示資料，例如每個人都必須知道的 30 個提示之類的文章。其中許多都集中在 chatGPT 網路使用者介面上，許多人使用它來執行特定的、通常是一次性的任務。但是，我認為 LLM、大型語言模型作為開發人員工具的強大功能（即使用 API 呼叫 LLM 來快速建立軟體應用程式）仍然未被充分重視。

事實上，我在 Al Fund（DeepLearning.ai 的姊妹公司）的團隊一直在與許多新創公司合作，將這些技術應用於許多不同的應用程序，看到 LLM API 能夠幫助開發人員快速建立應用程序，我感到非常興奮。因此，在本課程中，我們將與您分享您可以做的一些可能性，以及如何做到的最佳實踐。有很多內容需要講解。首先，您將學習一些軟體開發的最佳實踐，然後我們將介紹一些常見的用例，總結、推斷、轉換、擴展，然後您將使用 LLM 建立聊天機器人。我們希望這能夠激發您對可以建立的新應用程式的想像。因此，在大型語言模型或 LLM 的開發中，大致上有兩種類型的 LLM，我稱之為基礎 LLM 和指令調整 LLM。因此，基礎 LLM 已經過訓練，可以根據文字訓練資料預測下一個單詞，通常使用來自互聯網和其他來源的大量資料進行訓練，以找出下一個最有可能出現的單字。因此，例如，如果你提示我們從前有一隻獨角獸，它可能會完成這一點，也就是說它可能會預測接下來的幾個詞是和所有獨角獸朋友一起生活在一個神奇的森林裡。

但如果你要我們回答法國的首都是哪裡，那麼根據網路上的文章，基礎法學碩士很可能會回答法國最大的城市是哪裡、法國的人口是多少等等，因為網路上的文章很可能是關於法國這個國家的測驗問題清單。

相較之下，指令調整的 LLM 是 LLM 研究和實踐的大量動力所在，指令調整的 LLM 已經過訓練以遵循指令。因此，如果你問它法國的首都是哪裡，它很可能會輸出類似「法國的首都是巴黎」這樣的訊息。因此，指令調整的 LLM 通常的訓練方式是，首先使用已在大量文字資料上訓練過的基礎 LLM，然後進一步訓練它，使用指令的輸入和輸出以及遵循這些指令的良好嘗試對其進行進一步微調，然後通常使用一種稱為 RLHF 的技術（從人類反饋中進行強化學習）進一步改進，以使系統能夠更好地提供幫助並遵循指令。由於指令調整的 LLM 已被訓練為有用、誠實和無害的，因此與基礎 LLM 相比，它們不太可能輸出有害文字（例如有毒輸出），因此許多實際使用場景已轉向指令調整的 LLM。您在網路上找到的一些最佳實踐可能更適合基礎 LLM，但對於當今的大多數實際應用，我們建議大多數人專注於更易於使用的指令調整的 LLM，並且由於 OpenAl 和其他 LLM 公司的工作變得更加安全和更加一致。

因此，本課程將重點介紹針對指令調整的 LLM 的最佳實踐，我們建議您在大多數應用程式中使用它。在繼續之前，我只想感謝 OpenAl 和 DeepLearning.ai 的團隊為 Isa 和我將要展示的材料做出了貢獻。我非常感謝 OpenAl 的 Andrew Mayne、Joe Palermo、Boris Power、Ted Sanders 和 Lillian Weng，他們積極參與我們的頭腦風暴材料、審查材料以整理這門短期課程的課程，同時我還要感謝 DeepLearning 方面的 Geoff Lodwig、Eddy Shyu 和 Tommy Nelson 的工作。因此，當您使用經過指令調整的 LLM 時，請考慮向另一個人發出指令，例如說某個聰明但不知道您的任務具體內容的人。因此，當法學碩士學位不起作用時，有時是因為指示不夠明確。例如，如果你說，請寫一些關於艾倫圖靈的東西給我。嗯，除此之外，明確你想讓文章專注於他的科學工作、個人生活、歷史角色或其他方面，這會很有幫助。如果你指定了文本的基調，它是否應該採用專業記者寫作的基調。還是它只是你匆匆寫給朋友的便條？
