## Gradio
目的在於快速搭建一個模型的應用，讓使用者易於展示與操作。
* 只需要簡單幾行程式碼就快速搭建一個美美的網頁應用給別人看，完全不需要煩惱前端技術、同步非同步的問題，非常適合快速展示概念驗證。
* 在機器學習領域相當受歡迎的套件。
* 於 2019 年創立，由 Stanford 的 PhD Abubakar Abid 與幾位室友一起合作的專案，在 2021 年 12 月加入 Hugging Face 家族。

### gr.Blocks 基本用法
官方提供相當簡潔的 [Interface 類別](https://www.gradio.app/docs/interface)，可以用三行程式碼搞定一個基本介面。但如果要細緻一點的排版，通常會用到 Block 類別。

## 貓貓塔羅
透過 ChatGPT API + Gradio 搭建一個簡單的貓貓塔羅示範應用。

### 素材蒐集
1. 塔羅牌的牌面圖片，參考別人專案中裡面提供的 78 張[偉特塔羅牌](https://w.wiki/7Vt8)。
    * 但是抽塔羅牌時，會有正反向的問題，因此也需要準備另外 78 張旋轉 180 度的圖片。
2. [解牌資訊](https://gist.github.com/penut85420/e0d7dc33155c86c918375d09123f9c1d)

### 核心應用
* System Prompt 
    1. 先從「人設」著手，使用俗稱「催眠」的手段來進行
        > 你現在是一個專業的塔羅牌占卜師，而且你的身份是貓咪，所以你會使用很多「喵喵」做為句末助詞以及口頭禪。
    2. 需要進行的任務
        > 我會輸入一個問題，以及一張塔羅牌，你必須根據這張塔羅牌所代表的涵義，針對提出的問題給出詳細的解釋。
    3. 要求 ChatGPT 往積極樂觀的方向做解釋 (避免讓使用者心生陰影)
        > 在解釋問題時，請盡量往正面、積極的方向做解釋，並鼓勵對方。
    4. 要求 ChatGPT 隱藏自己的身份。另外我們只做單輪對話，所以也要避免 ChatGPT 企圖開啟第二輪對話。
        > 在這個過程中，你不能透露你是 AI，也不能透露你是語言模型，也不要提及你的身份，也不要向我要求更多訊息。
    5. 額外設定
        > 解釋完之後要用「喵喵解牌完畢！」做結尾。請使用繁體中文。
* User
    * 輸入占卜的相關資訊
        > 問題：我能穿越到異世界開掛嗎
        > 塔羅牌：正向愚者
        > 相關詞：天真、單純、可能、流浪、自由、隨興、古怪、輕浮、妄想、浪費、瘋狂、無知。

## 結論
在解讀塔羅牌時，並非單純只是照本宣科，依照解牌書逐字朗讀而已。很多時候會因為被占卜人的問題、生活、背景與個性，而有不同的解讀方式。但更重要的是，要為這些向塔羅牌求助的人點亮一盞指引方向的「明燈」。

ChatGPT並沒有辦法做到這麼深入，因為ChatGPT無法真正「認識」你。但如果能對使用者起到一絲絲一點點鼓勵的作用，那也算是有意義了。
